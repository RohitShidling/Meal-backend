BUUTTII MEAL BACKEND - SECURITY AND POSTGRESQL HARDENING REPORT
Generated On: 2026-05-07
Scope: Backend (Node.js/Express) + PostgreSQL schema/query/runtime hardening

====================================================================
1) WHAT WAS FOUND (CONSOLIDATED)
====================================================================

CRITICAL
- Payment webhook forgery risk (missing callback signature verification).
- Payment finalization race/idempotency corruption risk.
- SQL injection patterns in meal skip and analytics dynamic SQL.
- Admin OTP verification path not challenge-bound to password check.
- Pooled pseudo-transactions (BEGIN/COMMIT with db.query) risked partial commits.
- Runtime schema mutation/destructive migration behavior at server startup.

HIGH
- Stale cart price checkout (old prices paid after plan price change).
- Meal skip extension logic gave free meal-days (calendar vs eligible meal-days).
- OTP abuse controls were previously weak/non-distributed.
- Missing indexes on major hot paths (subscriptions/skips/orders/logs).
- Token/report endpoints had N+1 and synchronous heavy processing patterns.
- CORS/docs/rate-limit exposure posture needed tightening.

MEDIUM
- Query-string token acceptance in middleware (token leakage via URL).
- Offset pagination patterns on heavy endpoints.
- Some endpoints over-fetched data and had repeated expensive DB work.
- Error-message detail leakage risk in selected controllers.
- Polymorphic entity model has weaker DB-level integrity guarantees.

2) WHAT WAS FIXED (IMPLEMENTED)
====================================================================

2.1 PAYMENT SECURITY + INTEGRITY
Files:
- src/client/controllers/paymentController.js
- src/client/routes/paymentRoutes.js
- src/server.js

Implemented:
- Webhook signature validation with PhonePe callback verification.
- Raw-body handling for webhook route to verify signature correctly.
- Centralized success/failure state transition helpers.
- Transactional idempotent finalization flow with row locking.
- Added order finalization metadata (source/timestamp).
- Cart checkout now validates active plans + authoritative price before order creation.
- Cart checkout now locks cart/cart_items and runs atomically in transaction.
- Redirect allowlist sanitization for callback/front-end redirect URL.
- Reduced repeated expensive gateway calls by short TTL status cache.

2.2 SQL INJECTION + MEAL SKIP LOGIC
Files:
- src/client/controllers/mealController.js
- src/admin/controllers/subscriptionAnalyticsController.js

Implemented:
- Removed interval string interpolation, replaced with parameterized arithmetic.
- Meal skip extension now based on meal-eligible days (include_saturday aware).
- requestMealSkip and cancelMealSkip wrapped in DB transactions + locking.
- cancel flow made idempotent and tied to exact subscription row.
- analytics queries parameterized for schoolId/days/limit/offset.
- safer pagination parsing in analytics endpoints.

2.3 AUTH / OTP / JWT HARDENING
Files:
- src/admin/controllers/authController.js
- src/client/controllers/authController.js
- src/common/services/otpService.js
- src/admin/middlewares/authMiddleware.js
- src/common/middlewares/commonAuthMiddleware.js

Implemented:
- Removed refresh-secret fallback usage.
- Admin OTP challenge-binding added:
  - login issues challenge token after password verify.
  - verify-otp requires challenge token + phone + code.
- OTP anti-abuse controls implemented:
  - cooldown, expiry, max attempts, block window, daily limit.
- OTP state moved from in-memory map to PostgreSQL table (otp_sessions).
- Query-string token auth removed in middleware.

2.4 SERVER EXPOSURE HARDENING

Implemented:

- Runtime DB bootstrap guard:

2.5 POSTGRESQL HARDENING
